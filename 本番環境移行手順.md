# 消耗品管理システム 本番環境移行手順

## 前提条件

- 本番PCにDockerがインストールされ、起動していること
- 本番PCにデータベース（MySQL）が設定済みであること
- 本番PCでポート8504が使用可能であること
- アプリケーションのソースコードが準備されていること

## 1. アプリケーションファイルの転送

### 1-1. 必要なファイルを本番PCに転送

以下は、Gitを利用して本番PCにソースコードを配置する手順です。

### 1-1. リポジトリのクローン

本番PCの適切なディレクトリ（例: `/app`）で、Gitリポジトリをクローンします。

```bash
cd /app
# リポジトリURLは適切に変更してください
git clone <リポジトリURL> syomohin
```

### 1-2. 環境変数ファイルの作成

本番PC上で `.env` ファイルを作成します。
※ `docker-compose.yml` で `PRIMARY_DB_PASSWORD` 等を参照するため、`.env` にはアプリ固有の秘密鍵などを設定します。DB接続情報は `docker-compose.yml` の環境変数定義（または本番サーバーの環境変数）で制御されますが、`.env` に記載して読み込ませることも可能です。

```bash
cd /app/syomohin
cp .env.example .env
```

`.env` ファイルを編集します。

```env
# データベース接続情報
# docker-compose.yml 内で環境変数として渡されるため、ここで指定する必要性は低くなりましたが、
# ローカルテストや上書き用として残します。
INVENTORY_DB_NAME=inventory_db
# 共有DBのパスワード (必要であれば記載、またはシステム環境変数を使用)
PRIMARY_DB_PASSWORD=your_secure_password

# メール送信設定
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASSWORD=your-app-password
FROM_EMAIL=your-email@gmail.com
FROM_NAME=ダイソウ工業株式会社

# Flask設定
SECRET_KEY=your-production-secret-key-change-this
```

## 2. データベースの確認

本番環境の共有 MySQL コンテナ（ホスト名: `mysql`）に、データベース `inventory_db` が既に作成されていることを確認します。

```bash
# MySQLコンテナに入って確認する例
docker exec -it mysql mysql -u root -p -e "SHOW DATABASES LIKE 'inventory_db';"
```


## 3. アプリケーションの起動

`docker-compose.yml` を使用して起動します。
このファイルは既に共有ネットワーク `ts_pm_network_v2` を使用するように設定されています。

```bash
cd /app/syomohin

# ビルドと起動
# キャッシュを使わずにビルドする場合: --no-cache
docker-compose up -d --build
```

### 起動内容の確認（docker-compose.yml）

```yaml
version: '3.8'

services:
  app:
    build: .
    container_name: syomohin_app_prod
    restart: always
    ports:
      - "8504:8504"
    environment:
      - INVENTORY_DB_HOST=${PRIMARY_DB_HOST:-mysql}
      - INVENTORY_DB_PORT=${PRIMARY_DB_PORT:-3306}
      - INVENTORY_DB_USER=${PRIMARY_DB_USER:-root}
      - PRIMARY_DB_PASSWORD=${PRIMARY_DB_PASSWORD}
      - INVENTORY_DB_NAME=${INVENTORY_DB_NAME:-inventory_db}
      - FLASK_ENV=production
    volumes:
      - ./uploads:/app/uploads
      - ./data:/app/data
      - ./static:/app/static
      - ./.env:/app/.env:ro
    networks:
      - ts_pm_network_v2
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

networks:
  ts_pm_network_v2:
    external: true
```

## 4. 動作確認

### 4-1. コンテナの状態確認

```bash
docker ps | grep syomohin
# syomohin_app_prod が起動していることを確認
```

### 4-2. ログの確認

```bash
docker-compose logs -f
```

起動時にデータベース接続成功のログが出ているか確認してください。

### 4-3. ブラウザアクセス

```
http://本番PCのIPアドレス:8504
```

## 5. 運用コマンド

```bash
# 停止
docker-compose down

# 再起動
docker-compose restart

# ログ確認
docker-compose logs --tail=100 -f

# アプリケーションの更新（Git Pull & 再ビルド）
git pull
docker-compose down
docker-compose up -d --build
# 不要なイメージ削除
docker image prune -f
```

## 6. トラブルシューティング

### ネットワークエラー
`ts_pm_network_v2` が見つからないというエラーが出る場合、ネットワークが存在するか確認してください。

```bash
docker network ls
```

もし存在しない場合は、共有ネットワークを作成する必要があります（通常は他のアプリ起動時に作成されています）。

### DB接続エラー
ログに `Connection refused` や `Access denied` が出る場合：
1. `docker-compose.yml` の `environment` 設定値が正しいか確認。
2. 共有 MySQL コンテナが `ts_pm_network_v2` に参加しているか確認。
3. ユーザー名・パスワード・データベース名が正しいか確認。

### ヘルスチェック
簡易的な接続確認:
```bash
curl -I http://localhost:8504/login
```

## 7. バックアップ

定期的にバックアップを取ることを推奨します。

```bash
# データベースのバックアップ（MySQLの場合）
# コンテナ内またホストから mysqldump を実行
docker exec syomohin_app_prod mysqldump -h mysql -u root -p${PRIMARY_DB_PASSWORD} inventory_db > backup_$(date +%Y%m%d).sql

# アップロードファイルのバックアップ
tar -czf uploads_backup_$(date +%Y%m%d).tar.gz uploads/
```

## 8. セキュリティ設定（推奨）

### HTTPS対応

本番環境では、HTTPSを使用することを強く推奨します。

1. SSL証明書を取得（Let's Encrypt等）
2. Nginxなどのリバースプロキシを設定
3. Docker Composeで Nginx コンテナを追加

### ファイアウォール設定

```bash
# ポート8504への外部アクセスを許可（必要に応じて）
sudo ufw allow 8504/tcp
```

## 9. 監視とメンテナンス

### ログローテーション

Dockerログが肥大化しないよう設定します。

```bash
# docker-compose.ymlに追加
services:
  app:
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"
```

### ヘルスチェック

定期的にアプリケーションの稼働状態を確認します。

```bash
# ヘルスチェックスクリプト例
curl -f http://localhost:8504/login || echo "アプリケーションが応答しません"
```

---

以上で、本番環境への移行が完了です。
問題が発生した場合は、ログを確認して適切に対処してください。
